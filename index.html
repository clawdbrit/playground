The Box Model

Every element on a page is a box. This is what that box is made of.

The concept

When the browser renders an element, it wraps it in four nested layers: content, padding, border, and margin. They're always there, even if they're zero.

Content is the actual stuff — text, image, whatever's inside. Padding is the breathing room between the content and the border — it's inside the element, so it inherits background color. Border wraps the padding. You can see it. Margin is space outside the element — it pushes other things away. It's always transparent.

The confusing part: width and height in CSS don't always mean the same thing depending on a property called box-sizing. Drag the sliders below and watch what happens.

What this looks like in CSS
.my-element { box-sizing: border-box; width: 120px; height: 72px; padding: 20px; border: 4px solid #something; margin: 24px; }
/* rendered width: 120px (border-box — padding+border eat into the width) */
Why it matters

This is why your layouts break in weird ways. You set width: 100% on an input, add some padding, and suddenly it overflows its container. That's content-box biting you — the padding gets added on top of the width.

It's also why almost every CSS reset starts with * { box-sizing: border-box }. With border-box, padding and border eat into the width you set instead of expanding it. Your math works like you'd expect.

Margins don't get included in width either way — they live outside the element entirely. And they have a quirk: vertical margins collapse. Two stacked elements with margin-bottom: 20px and margin-top: 20px don't give you 40px of space — they give you 20px, because the larger one wins.
